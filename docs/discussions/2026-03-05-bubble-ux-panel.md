# 버블 미니 컨버터 헤더 UX 패널 토론

**날짜:** 2026-03-05
**참석자:** UX 디자이너(유진), 제품 전략가(민수), Flutter 개발자(세영), K-pop 팬 대리인(하늘)

## 배경

미니 컨버터(플로팅 버블 팝업)의 상세모드/간편모드 상단 영역 UX/UI 개선 논의.

**기존 문제점:**
- 상세모드 헤더: `[← 간편모드] ... [🔗 Fangeul] [X]` — "Fangeul" 텍스트 밋밋, `open_in_new` 아이콘 촌스러움
- 간편모드: 드래그 핸들 우측 작은 `open_in_new` 아이콘 — 역시 촌스러움
- 타겟: 10~20대 여성 K-pop 팬(한국, 동남아)

---

## 1차 토론: 브랜딩 + 메인앱 전환 + 핸들 구조

### 합의 (만장일치)
- **"Fangeul" TextButton 제거** — 유틸리티 팝업에 앱 이름 노출 불필요
- 핸들 pill → **틸(#4ECDC4)** 브랜드 컬러로 미니멀 브랜딩
- `open_in_new` 아이콘 → **`···` PopupMenuButton**으로 교체
- 두 모드: `[핸들바(틸)] + [···]` 구조

---

## 2차 토론: [← 간편모드] 버튼 제거

### 합의 (4:0)
- **제거** — 핸들 아래 드래그로 대체
- 상세모드 진입자 = 100% 핸들 학습 완료 상태 (올려서 진입했으므로)
- 바텀시트 표준 패턴 (Google Maps, Instagram, 카카오톡)과 일치
- 두 모드 동일한 핸들 제스처 = 일관성 향상

---

## 3차 토론: [X] 닫기 버튼 + "버블 닫기" 메뉴

### 핵심 발견
"팝업 닫기"와 "버블 서비스 종료"는 **다른 동작**:

| 동작 | 코드 | 결과 |
|------|------|------|
| X / 핸들 드래그 | `SystemNavigator.pop()` | 팝업만 닫힘, 버블 떠있음 |
| `···` "버블 닫기" | `stopService()` | 버블 서비스 완전 종료 |

### 사용자 지적
> "버블 닫기를 누르면 서비스가 완전 종료되고, 다시 키려면 메인앱→설정→토글. 사용자가 의도한 건 그게 아님."

### 합의 (4:0 만장일치)
- **[X] 버튼 제거** — `···` 메뉴의 "팝업 숨기기"가 대체
- **"버블 닫기" 메뉴 제거** → **"팝업 숨기기"**로 교체
  - "팝업 숨기기" = `SystemNavigator.pop()` (버블 유지)
  - 버블 서비스 종료는 기존 경로 유지: 노티피케이션 액션 / 메인앱 설정
- 미니 컨버터는 "잠깐 쓰고 닫는" 도구 — 서비스 종료 경로가 여기 있을 이유 없음
- 실수로 "버블 닫기" → 재시작 비용(설정→토글) 너무 큼 → DAU 이탈 위험

---

## 최종 설계

```
간편모드: [핸들바(틸)] ............ [···]
상세모드: [핸들바(틸)] ............ [···]

핸들 드래그:
  간편 아래 → 팝업 숨기기(버블 유지)
  간편 위로 → 상세모드 전환
  상세 아래 → 간편모드 복귀

··· 메뉴:
  ├─ Fangeul 앱 열기
  └─ 팝업 숨기기 (버블 유지)

버블 서비스 종료 (기존 경로 유지):
  ├─ 노티피케이션 → "버블 끄기" 액션
  └─ 메인앱 설정 → 버블 토글 OFF
```
