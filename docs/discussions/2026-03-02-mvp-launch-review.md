> **날짜**: 2026-03-02
> **상태**: 보완본 (v2) — 추가 논의 + 리서치 + Codex 리뷰 반영
> **참석 전문가**: 민준(동남아 PM), Sarah(인디 개발자), 재현(K-pop 마케팅), David(VC), 수빈(UX), Priya(실사용자)

## 논제

현재 구현 상태(Phase 1~5)로 MVP 충분한가? 출시 타이밍, 경쟁 방어력, 리텐션, 수익화.

---

## Topic 1: 현재 기능셋 MVP 충분성

**합의**: 핵심 기능 루프(키패드+변환+발음+문구+버블) 완성. Play Store에 이 조합을 가진 앱 없음.
**핵심**: 플로팅 버블이 유일한 구조적 차별점 — "앱 위의 레이어"로 대체재(구글 번역, Papago)와 다른 카테고리.
**쟁점**: D7 예측(15~25% vs 5~10%), "one-time task" 문제(문구 3-5개 복사 후 미복귀).

## Topic 2: 출시 타이밍 & 경쟁 방어력

**합의**: Closed Beta(300명)가 타협점. 핵심 3개 플로우(입력, 변환, 버블) 크래시 제로 필수.
**핵심**: 30초 온보딩에서 "변환 한 번 + 버블 한 번" 핵심 감동 포인트 2개 전달이 성패 가름.
**경쟁 분석**: Gboard 등 메가앱의 기능 추가 위협은 12-18개월 소요 → 선점이 "유통 moat" 형성.
**쟁점**: Signal Risk(미완성 노출 vs 타이밍 상실), 1점 리뷰 영구 잔존 리스크.

## Topic 3: 리텐션 & 실사용 시나리오

**합의**: DAU를 "앱 오픈"이 아닌 "버블 세션"으로 재정의. "복사 후 자동 닫힘" UX가 핵심 moat.
**핵심**: 버블은 습관 루프 Trigger를 앱 외부에 두되, 화면 위에 상시 존재하여 루프 내부화 불요.
**팬 행동**: SNS 댓글 루틴(일상) + K-pop 이벤트(생일/컴백, 스파이크) + 소셜 인정(팬력) 3중 동기.
**현실적 D7**: 버블 Permission 허용 유저 기준 15~20%, 전체 설치 기준 8~12%.
**쟁점**: 게임화 없이 D30 유지 가능한가, "변환기=창피함" 프레이밍 문제.

## Topic 4: 수익화 타이밍 & 동남아 시장

**합의**: 기본 기능 무료 + 감성 팩 IAP 전원 합의. 시나리오 검증이 수익화보다 선행.
**핵심**: 동남아 신용카드 6-7%, 통신사 결제/전자지갑 의존. "유료=공유 차단" 아님 — 소셜 노출 루프가 핵심.
**타이밍**: D7 10%↑이면 수익화 선행(Sarah), D7 3%↓이면 후행(민준). 데이터로 결정.
**쟁점**: "팬 패스"가 리텐션 도구인가 지연된 페이월인가.

## Topic 5: 문구 전략 — 양보다 맥락 (추가 논의)

### 리서치: 동남아 K-pop 팬 채팅 행동

| 발견 | 데이터 |
|------|--------|
| 90% 고정 구문 복사-붙여넣기 | "화이팅!", "사랑해요!", "생일 축하해요!" 5~10개 반복 |
| 50% 1-3단어 단문 | 감정 전달 최우선, 문법 부차적 |
| 현재 해결책 | Koreaboo/Twitter 구문 목록 북마크 → 복사 → 붙여넣기 |
| 주 플랫폼 | Twitter/X > Weverse > TikTok > Instagram > LINE(태국) |
| KakaoTalk | 팬덤 소통에 거의 사용 안 함 (한국 내부용) |
| 팬덤 믿음 | 한국어 댓글이 아이돌에게 읽힐 확률 높다 |
| 팬챈트 수요 | 콘서트 전 응원법 예습 — 멤버 이름 나열 순서 |

출처: Koreaboo, Weverse Fandom Report 2024, UCLA 언어학 연구, 인도네시아 K-pop 팬 다국어 연구

**전원 합의**: 150개 확장은 불필요. 80개도 '상황 진입점' 없으면 사용 안 됨.
**핵심**: **K-pop 캘린더 연동 + 상황 자동 서페이싱**이 문구 양보다 10배 가치.
- "오늘 XX 생일" → 생일 문구 3개 즉시 서페이싱
- 상황 태그(`birthday/comeback/concert/daily/support`) 기존 80개에 추가
- 탐색 비용 0: 홈에서 탭 2번 안에 오늘 쓸 문구 나와야 함

## Topic 7: 버블 UX — 하단 시트 비율 & 엣지 (추가 논의)

**현재**: 간편모드 30%, 확장모드 70%.

**전원 합의**: **30% → 43%로 상향.**
- 30%: 드래그 핸들+탭바+필터칩+카드가 빡빡하게 배치, 카드 내 텍스트 여유 부족
- 43%: 필터 칩 완전 노출 + 카드에 숨 쉴 공간 확보 (카드 탐색은 좌우 PageView 스와이프)
- 화면 상단 57% 이상 원래 앱 유지 → 오버레이 가벼움 보존
- 50% 초과는 반쪽 앱 느낌 → 하드 상한

**확인된 UX 강점**:
- "복사 후 자동 닫힘" = 핵심 moat (Weverse, 구글 번역 어디에도 없음)
- 버블 = 앱 전환 없이 2초 완료

**개선 우선순위**:
1. 마지막 사용 팩 자동 복원 → 첫 복사 5탭→3탭
2. confetti + 진동 복사 피드백
3. 카드 배경에 감성 컬러 프리뷰 (IAP 전환 퍼널)

## Codex 독립 리뷰 (GPT-5.3-Codex)

> "제품 아이디어는 강하지만, 1인 팀에 비해 계획이 MVP-clean하지 않다."

### 패널이 놓친 블라인드 스팟

| 지적 | 심각도 | 대응 |
|------|--------|------|
| "DAU = 버블 세션"은 범주 오류 — DAU는 유저 수, 세션은 행동량. 둘 다 필요 | 중 | DAU(유니크 유저) + 버블 세션(빈도) 이중 지표로 수정 |
| "크래시 제로"는 실용적 기준 아님 → `>99.5% crash-free sessions` by device tier | 중 | Crashlytics device tier 분리 대시보드 |
| "copy-then-auto-dismiss가 moat"는 검증 안 됨, 내구성 없음 → 실행 품질이 진짜 moat | 고 | moat를 UX 특허가 아닌 실행 속도+커뮤니티 침투로 재정의 |
| "Context > quantity" 맞지만 컨텍스트 구축 비용을 과소평가 | 고 | K-pop 캘린더 MVP는 manual curated JSON으로 축소 |
| 수익화 로직 순환: D7로 IAP 결정하려면 pre-launch 리텐션 증거 필요, 근데 Beta가 대표성 있나? | 중 | Beta → Open 전환 시 D7 재측정 후 결정 |

### 패널이 충분히 다루지 않은 리스크

| 리스크 | 대응 방안 |
|--------|----------|
| **OEM 배터리 최적화** — 버블 서비스 강제 종료 (Xiaomi, Samsung, Oppo) | `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` + 온보딩 안내 |
| **Google Play 오버레이 정책** — 사용자 신뢰 리스크 | Play Console 정책 준수 체크리스트 작성 |
| **메모리/배터리 드레인** — 키보드 + 오버레이 + 백그라운드 서비스 | Foreground Service 알림 유지, 미사용 시 자동 절전 |
| **한국어 뉘앙스 오류** — 팬덤 맥락에서 신뢰 즉시 파괴 | 네이티브 검수 필수 (출시 전) |
| **분석 사각지대** — 복사 성공/실패/지연 미계측 시 핵심 루프 최적화 불가 | 복사 이벤트 + 지연시간 + 실패 사유 계측 |
| **보상형 광고 = 오버레이에서 스팸 느낌** — 리텐션 해침 | 보상형은 메인 앱에서만, 버블에서 제외 |

### K-pop 캘린더 — MVP 현실성

> "풀 자동화는 솔로 MVP에 비현실적. 수동 큐레이션 + remote config JSON이 현실적."

**Codex 권장 MVP 버전 (수정):**
- 공개 K-pop 스케줄/위키 크롤링 또는 캘린더 MCP → Claude Code CLI(Max 구독)로 정제 → `assets/calendar.json` 일관된 포맷 생성
- 앱 시작 시 오늘 날짜 매칭 → 해당 상황 태그 문구 서페이싱
- 데이터 갱신: 월 1회 파이프라인 실행 → 앱 업데이트 or Remote Config
- **풀 자동 인앱 크롤링은 Phase 7+로 연기, 빌드 타임 반자동화는 MVP에 포함**

### 43% 하단 시트 — 추가 UX 우려

| 우려 | 대응 |
|------|------|
| 작은 화면 + 큰 폰트 시 43%에서도 칩/액션 잘림 | adaptive height by screen class 고려 |
| 가로 모드/분할 화면에서 어색 | 가로 모드 미지원 (Android only, portrait lock) |
| 엄지 도달 범위 위로 밀림 | 핵심 액션(복사)은 카드 내 하단 배치 |
| 반복 세션에서 43%가 무겁게 느껴질 수 있음 | 복사 후 즉시 닫힘으로 노출 시간 최소화 |

### Codex 최종 권장: "이것만 하고 출시하라"

| 유지 (MVP) | 연기 |
