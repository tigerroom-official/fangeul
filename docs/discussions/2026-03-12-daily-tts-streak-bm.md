# 일일카드 TTS + 스트릭 BM 설계 — 전문가 패널 토론 기록

> **날짜**: 2026-03-12
> **참가자**: 서유진(PM), 민하린(UX), 장태호(게이미피케이션), 박지은(인디 개발자), 김소연(에듀 PM)

---

## 배경

- 문구 화면 TTS는 일단 숨기고, 매일 뜨는 일일카드 1장에만 "오늘의 한국어 발음 듣기" TTS 버튼 추가
- 무료 매일 1회 TTS 체험 → 핵심 가치(발음 학습) 맛보기 → 이후 전체 TTS를 프리미엄 IAP
- 현재 스트릭은 숫자만 쌓임 → 성취감/소속감/보상 연결 없음
- 컬렉션/카드 시스템, 마일스톤 보상, 스트릭 프리즈 등 BM 방향 탐색

## 제약

- 가사 표시 금지 (KOMCA), 아이돌 이름/이미지/로고 직접 사용 금지 (초상권+상표권)
- 팬덤명 직접 사용 금지, 특정 그룹 컬러 조합 판매 금지
- `assets/audio/`에 현재 실제 오디오 파일 0개 (`.gitkeep`만)
- `Phrase` 엔티티에 `audioUrl` 필드 없음

## Topic 1: 일일카드 TTS — 매력적인 엣지가 될 수 있는가?

### A조 (지지): 서유진, 민하린

**서유진 (PM):**
> "데이터로 판단합시다"

- 핵심 주장: "오늘의 한국어 발음 듣기"는 **매일 방문의 이유**를 만듦
- 근거: 경쟁 앱(키보드/변환기)에 TTS 없음 → 차별화. 팬 인사 발음 학습 = 실제 니즈 (콘서트 응원, SNS 코멘트)
- 보상형 광고 아닌 직접 가치 체험 → "이게 좋으니까 더 사고 싶다" 전환

**민하린 (UX 리서처):**
> "K-pop 팬에게 '한국어 한마디'는 최애와의 심리적 거리를 좁히는 행위"

- "팬 인사 연습" 프레이밍: "한국어 배우기" → "내 최애에게 인사 연습"
- 매일 1문구만 집중 → 과부하 없이 성취감. 전체 TTS는 나중에 IAP.

### B조 (우려): 박지은, 장태호

**박지은 (인디 개발자):**
> "현실부터 봅시다"

- **치명적 발견**: `assets/audio/`에 실제 오디오 파일 0개. `Phrase` 엔티티에 `audioUrl` 필드 없음.
- 콘텐츠 파이프라인 비용: 96개+ 문구 × 고품질 오디오 생성 = 상당한 작업량
- 당장 기술적으로 "Daily TTS 듣기"를 만들 수 없는 상태

**장태호 (게이미피케이션):**
> "습관 루프에서 보상이 없으면 루프 자체가 돌지 않아요"

- TTS 1회 무료 → IAP 전환 퍼널이 너무 김 (매일 1회면 충분히 만족)
- "1회 무료 → 더 듣고 싶다"가 작동하려면 여러 문구 TTS를 원하는 상황이 와야 함

### A조 재반론

**서유진:** 전체 TTS IAP가 아니라 **번들 통합**이 답. 기존 ₩1,500 번들에 TTS 포함 → ₩1,900으로 업그레이드.
**민하린:** 오디오 30개(basic_love 팩)부터 시작 → 사용자 RTX 5090 서버로 TTS 생성 가능.

### 사회자 중간 정리

**합의:**
- 일일카드 TTS는 "팬 인사 연습" 프레이밍으로 매력적인 엣지가 될 수 있음
- 단, 콘텐츠 파이프라인(오디오 생성)이 선결 과제
- 별도 SKU 대신 기존 번들에 통합 (3-SKU 유지, 번들 가격만 조정)

**쟁점:**
- 30개 오디오로 충분한가? 매일 카드가 같은 오디오를 반복하면?
- 번들 ₩1,900 가격 적정성 (동남아 기준)

## Topic 2: 스트릭 시스템 — K-pop 팬에게 유효한 습관 루프인가?

### A조 (지지): 장태호, 김소연

**장태호 (게이미피케이션):**
> "Duolingo 스트릭이 작동하는 진짜 이유는 '내가 쌓아온 것을 잃기 싫은' 손실 회피"

- K-pop 팬 심리: "매일 최애를 위해 뭔가 하는 나" = 팬 정체성 강화
- 마일스톤 메시지 (7일, 30일, 100일)만으로도 감성적 보상 효과
- "진정한 팬은 매일 한마디" — 소속감 프레이밍

**김소연 (에듀 PM):**
> "학습과 팬덤의 교차점이 이 앱의 핵심 가치"

- Tiny Habits 모델: "카드 보기 → Done 누르기"는 30초 미만의 초소형 습관
- 외재적 보상(아이템) 없어도 내재적 보상(성장 실감)으로 유지 가능
- 단, "Done 누르기"만으로는 학습 깊이 부족 → TTS 듣기가 보강

### B조 (우려): 서유진, 민하린

**서유진 (PM):**
- 스트릭 피로: Duolingo조차 "스트릭 압박" 비판 (유저 이탈 원인)
- K-pop 팬 = 다중 덕질 (여러 앱/플랫폼) → Fangeul만 매일 여는 것은 비현실적일 수 있음
- 스트릭 프리즈 유료화? → **반대**. 스트릭을 통제의 도구로 쓰면 팬 이탈

**민하린 (UX 리서처):**
- "매일 앱 열기"와 "매일 한국어 한마디"는 다른 프레이밍
- "앱 출석"은 의무감 → "오늘의 한국어 한마디"는 자축(celebration)
- 스트릭 깨져도 벌칙 없어야 함 (리셋만, 추가 손해 없음)

### A조 재반론

**장태호:** "자축" 프레이밍 동의. "N일 연속 한국어 도전 중! 🎉" — 의무가 아닌 자랑거리로.
**김소연:** 스트릭 프리즈는 무료로 제공 (기존 `freezeCount` 필드 활용). 유료 프리즈는 안 함.

### 사회자 중간 정리

**합의:**
- 스트릭은 "자축" 프레이밍으로 유지 (의무/벌칙 아님)
- 마일스톤 메시지는 감성적 보상으로 유효
- 스트릭 프리즈는 무료 (유료 판매 안 함)
- 스트릭 → 직접 수익화(유료 프리즈)는 하지 않음

**쟁점:**
- 마일스톤에서 테마 체험권 연결할 것인가?
- 스트릭 숫자 공유 기능 필요성

## Topic 3: 마일스톤 → 보상형 연결 — 수익화 경로로 적합한가?

### A조 (지지): 서유진, 장태호

**서유진 (PM):**
- 마일스톤 도달 시 "축하! 프리미엄 테마 24h 체험권" → 자연스러운 전환 퍼널
- 보상형 광고 시청 없이 **무상 체험** → "이거 좋네, 사야겠다"
- RC 플래그(`streak_milestone_trial`)로 기본 OFF, A/B 테스트 후 결정

**장태호 (게이미피케이션):**
- "노력의 보상"으로 포장하면 over-justification effect 회피
- 7일 → 24h 체험, 30일 → 48h 체험 (단계적 확대)

### B조 (우려): 박지은, 김소연

**박지은 (인디 개발자):**
- 마일스톤 체험이 오히려 IAP 전환을 **감소**시킬 수 있음 — "기다리면 또 공짜로 주겠지"
- 체험 중 만족 → 끝나면 그냥 떠남 (체험 피로)
- 스트릭 마일스톤은 "축하 메시지"에 그쳐야 함. 물질적 보상 연결은 위험

**김소연 (에듀 PM):**
- 교육 앱에서 "무료 체험→전환"은 보통 5~8% — 대부분은 체험 후 이탈
- "축하 메시지 + 간접적 테마 프리뷰(스크린샷)"가 더 나을 수 있음

### A조 재반론

**서유진:** RC 플래그로 기본 OFF → 데이터 확인 후에만 활성화. 리스크 최소화.
**장태호:** A/B 테스트 없이 판단 불가. 둘 다 맞을 수 있음. 플래그만 심자.

### 사회자 중간 정리

**합의:**
- RC 플래그(`streak_milestone_trial: false`) 심어두되 기본 OFF
- MVP에서는 마일스톤 = 축하 메시지만 (체험권 미연결)
- A/B 테스트 후 데이터 기반 결정

## Topic 4: 카드 컬렉션 & StarDust — IP 제약 하에서 실현 가능한가?

### A조 (지지): 민하린, 장태호

**민하린 (UX 리서처):**
- 팬 한국어 표현 + 최애 색상 + `{{group_name}}` 템플릿 = IP-safe 팬 정체성 카드
- "내가 모은 한국어 표현 컬렉션" = 포켓몬 도감 심리
- 희소성: 스트릭 유지 시 레어 등급 드롭

**장태호 (게이미피케이션):**
- StarDust = 소프트 통화. 스트릭으로 적립, 카드 등급업에 소모
- 가챠가 아닌 "선택적 수집" (IP 리스크 없음)

### B조 (우려): 서유진, 박지은

**서유진 (PM):**
- Phase 7+ 기능. MVP에 넣으면 스코프 폭발
- StarDust 경제 밸런싱은 별도 설계 필요 (인플레이션, 싱크/소스)

**박지은 (인디 개발자):**
- UserProgress에 `collectedCardIds`, `starDust` 이미 선언됨 → 확장 가능
- 하지만 카드 메타데이터 스키마, 렌더링 위젯, 컬렉션 화면 전부 신규 → 큰 작업

### 사회자 중간 정리

**합의:**
- 카드 컬렉션 + StarDust = **v1.1로 연기** (사용자 결정)
- UserProgress 필드는 이미 준비됨 → 확장 비용 낮음
- MVP는 스트릭 + 축하 메시지에 집중
