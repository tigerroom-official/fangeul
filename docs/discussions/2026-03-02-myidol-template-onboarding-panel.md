## Topic 1: 온보딩 + 마이 아이돌 첫 선택 플로우

### 합의 사항 (전원)

- **단일 그룹 선택**: 멀티스탠 지원은 Phase 7+. MVP는 "지금 가장 좋아하는 그룹 하나" 프레이밍
- **5그룹 제한 철폐**: 사전 정의 리스트(초기 5그룹 파일럿) + "기타/직접 입력" 탈출구 필수
  - 직접 입력은 서버 미검증(보안 불필요) — 유저 로컬에서만 사용하는 문자열
- **설정 변경 상시 고지**: "설정 > 마이 아이돌에서 언제든 변경 가능" 문구 온보딩 및 설정 화면에 노출
- **화면당 결정 수 = 1개**: 인지 부하 최소화. 한 화면에 하나의 선택만

### 쟁점 A: 온보딩 단계 수

**A조 (윤하나, 정민수) — 3단계 지지:**
- Duolingo 사례: 3단계 온보딩 D1 리텐션 +22%
- "앱을 이해하지 못한 채 메인 진입" → 첫 세션 이탈 주원인
- 단계: ①환영+가치 제안 → ②아이돌 선택 → ③버블 체험

**B조 (박실행, Sarah Kim) — 1단계 지지:**
- SEA 저가 기기 3단계 = 12~15초 추가 → Permission 요청 전 이탈 3~5%
- "돌아가서 선택 안 함"이 최대 리스크: 직접 메인 + 인라인 툴팁이 이탈 방어
- Fangeul은 설명이 필요 없을 정도로 즉각적인 앱이어야 함

**이수진 — 중립:**
- 법적으로는 어떤 구조든 무관. 단, 아이돌 선택 화면에 "이름은 서버에 저장되지 않음" 표기 권고

### 쟁점 B: 변환/버블 체험 시점

**A조 (윤하나, 정민수):** 온보딩 내 미니 체험 필수 — "직접 해본 사람"과 "설명만 읽은 사람"의 D7은 2배 차이 (앱 카테고리 평균)

**B조 (박실행, Sarah Kim):** 인라인 툴팁 + 코치마크 — 온보딩 길어지면 Permission 요청 전 이탈. 복사 성공 순간이 진짜 "wow"이므로 가로막지 않는 것이 최선

### 쟁점 C: 건너뛰기 제공 여부

**윤하나, 정민수:** 아이돌 선택 없이 진입하면 앱의 맥락형 문구가 전혀 작동 안 함 → "건너뛰기" = 반쪽 경험 강요. 미제공이 낫다.

**박실행:** 강제 온보딩 = 4.x 스토어 리뷰에서 "억지로 선택해야 함" 악평 패턴. 탈출구는 열어두되 "나중에 설정하기" 레이블로 심리적 부담 낮추기.

**Sarah Kim:** 건너뛰기 유저 = 어차피 비활성. 막는 비용 > 막는 이익. 제공하되 설정 진입점 재강조.

## Topic 2: 마이 아이돌이 앱 전체에 미치는 UX 변화

### 합의 사항 (전원)

- **공식 컬러 직접 적용 금지**: 퍼플 드림·골든 아워 등 감성 컬러 팩으로만 연결 (IP + rules/00-project.md 준수)
- **SharedPreferences 저장**: `my_idol_group` 키, 암호화 불필요 (보안 위협 없음)
- **캘린더 필터 기본값 연동 = MVP 포함**: 마이 아이돌 설정 완료 시 캘린더 탭 기본 필터를 해당 그룹으로 자동 설정
- **감성 컬러 팩 소프트 추천**: "XX 팬에게 어울리는 컬러 팩" 형태로 매핑만. IAP 전환 퍼널로 연결

### 쟁점 D: 홈 변화 범위

**A조 (윤하나, 정민수) — 이벤트 인라인 칩 포함:**
- "오늘 XX 생일" 칩이 홈에 있어야 캘린더 탭 진입 동기 생김
- 칩 없으면 캘린더 탭은 유저가 능동적으로 찾아야 → 발견율 20% 이하

**B조 (박실행, Sarah Kim) — 인사말 한 줄만:**
- 홈 UI 변경 = MVP 범위 초과 리스크
- 상황태그 + 캘린더는 이미 완성. UI 연결은 v1.2 빠른 업데이트에서
- 인사말 한 줄 ("안녕하세요, [그룹명] 팬!") = 0.5일 구현

### 쟁점 E: "오늘 추천" 배너 MVP 포함 여부

**A조 (정민수, 윤하나):** 배너가 진입점 없으면 맥락형 문구 발견율 급감. KpopEvent 매칭 로직이 이미 완성 — UI 연결만 남음.

**B조 (박실행, Sarah Kim):** 필터 칩으로 상황 진입을 강제하는 것이 더 검증된 패턴. 배너 = 별도 A/B 테스트 필요.

### 쟁점 F: 데일리 카드 가중 풀 범위

**정민수:** 마이 아이돌 설정 시 생일/컴백 팩 가중치 +50% (같은 상황 태그 내에서)

**박실행:** 가중 로직 = 오버엔지니어링. 상황태그 필터로 충분. 가중치는 데이터 쌓인 후 v1.3에서.

**Sarah Kim:** 동남아 유저는 "랜덤 추천"에 놀라지 않음. 필터 칩이 더 직관적.

## Topic 3: 템플릿 문구와 기존 문구의 공존

### 합의 사항 (전원)

- **`Phrase.isTemplate: bool` 추가**: 별도 entity 생성 없음. 기존 Phrase에 필드 추가 (Sprint 2 패턴과 동일)
- **즐겨찾기 키 = 원본 템플릿 문자열**: `{{group_name}} 화이팅!!`을 키로 저장 — 치환 후 저장 시 그룹 변경 시 깨짐
- **미설정 유저 = 템플릿 문구 미노출**: `myIdolProvider.value == null` 시 `isTemplate: true` 문구 필터링
- **치환 시점 = Provider 레이어**: `phrase_providers.dart`에서 `{{group_name}}` → `myIdolProvider.value` 치환. Widget에서 치환 금지

### 쟁점 G: 혼합 vs 분리 표시

**A조 (윤하나, 정민수) — 기존 팩 내 조건부 삽입 (혼합):**
- "마이 아이돌 설정 전/후 같은 팩을 보는 경험" = 설정 유도 자연 트리거
- 분리 칩은 UI 복잡도 증가

**B조 (박실행, Sarah Kim) — 별도 필터 칩 (분리):**
- 혼합 시 미설정 유저 팩 내에 빈 슬롯 또는 플레이스홀더 필요 → 콘텐츠 홀 문제
- 칩 방식: "내 아이돌" 칩 탭 시에만 노출, 명확한 진입점

**이수진 — 분리 지지:**
- 혼합 방식에서 템플릿 문구가 "앱이 아이돌 이름 사용"으로 오해받을 가능성 줄이려면 별도 구역이 안전

## Topic 4: 버블 내 맥락형 문구 서페이싱

### 합의 사항 (전원)

- **미설정 유저 "오늘" 칩 미표시**: 설정 완료 유저에게만 노출. 미설정 시 칩 자체 숨김 (빈 상태 없음)
- **KpopEvent.artist/group = 매칭 전용**: `todaySuggestedPhrasesProvider` 내부 로직에서만 사용. 버블 UI에 아티스트명 직접 노출 금지
- **마이 아이돌 Provider = cross-engine sync 필수**: 듀얼 FlutterEngine 환경에서 메인 앱과 버블 엔진 간 SharedPreferences `reload()` 패턴 적용 (기존 교훈 적용)
- **칩 방식 (0dp 추가)**: 새 탭 없이 기존 필터 칩 영역에 "오늘" 칩 추가. 레이아웃 변경 없음

### 쟁점 H: cross-engine sync 완성 전 블로킹

**박실행:** 마이 아이돌 sync 없으면 버블에서 "오늘 칩"이 항상 비어있음. 배포 순서: ①메인 앱 마이아이돌 UI → ②sync 구현 → ③버블 칩 노출. 블로킹이 맞다.

**Sarah Kim:** 동의. 단, sync 구현은 `prefs.reload()` 패턴이므로 추가 공수 소규모.

**윤하나:** 버블에서 칩 비어있는 상태 노출이 UX 손상. 블로킹 지지.

## 구현 의존 관계

```
[의존성 그래프]

A: MyIdolNotifier + SharedPreferences 저장
    ↓
B: groups.json lite (5그룹 데이터)
    ↓
C: 마이 아이돌 선택 UI (온보딩)
    ↓
D: Phrase.isTemplate 필드 + Provider 치환 로직
    ↓
E: 즐겨찾기 키 = 원본 템플릿 문자열 보장
    ↓
F: "내 아이돌" 필터 칩 (메인 앱)
    ↓
G: 미설정 유저 필터링 (isTemplate 문구 숨김)

A → H: cross-engine sync (prefs.reload())
    ↓
I: 버블 "오늘" 칩 (설정 시에만 표시)

A + K-pop 캘린더(완료) → J: 캘린더 필터 기본값 연동

A → K: 홈 인사말 한 줄
```

**구현 순서 (직렬 의존성):**
1. `MyIdolNotifier` + `groups.json` (A+B) — 모든 기능의 기반
2. `Phrase.isTemplate` + Provider 치환 (D+E) — 템플릿 시스템 기반
3. 마이 아이돌 선택 UI (C) — 의존: A+B
4. "내 아이돌" 필터 칩 + 미설정 필터링 (F+G) — 의존: D+C
5. cross-engine sync + 버블 칩 (H+I) — 의존: A
6. 캘린더 연동 + 홈 인사말 (J+K) — 의존: A, 병렬 가능
