## 핵심 합의

### 1. 앱 정체성
- **"K-pop 팬의 한국어 동반자"** — 유틸리티도 게임도 아닌, 팬 활동의 일부
- Core Action: **"아이돌에게 한국어로 말 걸기"** — 모든 기능이 이 목적으로 수렴

### 2. MVP 킬러 피처
- **플로팅 버블 + 문구 라이브러리** — 경쟁자 없는 유일무이 포지셔닝
- 핵심 시나리오: Weverse/YouTube 라이브 → 버블 탭 → 문구 복사 → 채팅 붙여넣기

### 3. MVP 리텐션
- **스트릭** (구현 비용 낮음, 앱 오픈 1회/일 = 유지)
- **"오늘의 문구" 카드 이미지 공유** — Acquisition(바이럴) + Retention(매일 새 카드) 동시
- 퍼즐/카드컬렉션/캘린더는 v1.1~v1.2로 연기

### 4. 기술 전략
- **서버리스 유지** (비용 0, 속도, 개인정보)
- **외부 플랫폼(X/Instagram) 공유가 소셜 대체** — 앱 내 커뮤니티는 시기상조
- **데이터 구조는 1년 비전, 기능은 3개월 PMF 단위**

### 5. 카드 비주얼
- MVP: **Flutter CustomPainter/애니메이션 기반 동적 생성** (한글 폰트+그라데이션+파티클)
- v1.1: ComfyUI 배치 생성으로 전환 검토 (사용자 피드백 기반)

## 토론 상세 (5개 토픽)

### Topic 1: 앱의 정체성
- 합의: "또 하나의 유틸리티"로는 생존 불가. Core Action은 기능이 아니라 감성/목적
- 쟁점: 기능 간 연결성(coherence) 설계가 핵심

### Topic 2: MVP 킬러 피처
- 합의: 플로팅 버블이 유일무이 차별화. 세션 연장 요소 필요
- 쟁점: gate(강제 학습) vs nudge(자연스러운 유도)

### Topic 3: 리텐션 전략
- 합의: Wordle/Duolingo 원리는 유효하나 K-pop 맥락 번역 필요
- 합의: "오늘의 문구 카드 이미지 공유"가 퍼즐보다 우선순위 높음
- 쟁점: 카드 비주얼 코드 생성 vs ComfyUI 전환 시점

### Topic 4: 서버리스 vs 소셜
- 합의: MVP 서버리스 유지. 외부 플랫폼 공유가 소셜 대체
- 합의: 데이터 구조에 소셜 확장 필드 미리 확보
- 쟁점: v1.2에서 Firebase vs 외부 연동 지속

### Topic 5: 1년 후 비전
- 합의: "팬 활동 기록(Fan Activity Log)" 중심. 레벨 시스템 X
- 합의: 코드는 3개월, 데이터 구조는 1년 단위
- 쟁점: 대기업(HYBE/SM) 유사 기능 출시 시 방어
