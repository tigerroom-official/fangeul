## 핵심 합의 사항

### 1. 맥락형 문구는 엣지가 맞다 (전원 합의)
- "내 아이돌 이름이 들어간 문구"는 기존 범용 문구와 본질적으로 다른 가치
- 매주 새 문구로 리텐션 루프 생성 가능
- 단, 전체 시스템을 한 번에 구축하지 않고 **단계적으로**

### 2. 템플릿 방식 이름 삽입 (전원 합의)
- AI가 "카리나 1위 축하해요"를 직접 생성하지 않음
- `{{group_name}} 1위 축하해요!!` 형태 템플릿 → 유저의 마이 아이돌 설정에 따라 런타임 삽입
- 법적으로 더 안전 (앱이 아이돌 이름을 사용하는 게 아님)
- 비용 절감 (그룹×멤버별 문구 생성 불필요)

### 3. 주간 → 월간 업데이트 (전원 합의)
- "금주의 문구" → **"이달의 문구"**
- 1인 개발자 운영 현실: 주간 검수(30분/주)는 맥락 전환 비용 포함 시 부담
- 월간: 1시간/월로 충분, 품질에 더 집중 가능

### 4. 규칙 수정 필요 (전원 합의)
- 기존: "아이돌 이름/이미지/로고 직접 사용 금지"
- 수정: "유저 선택 기반 개인화 내 이름 사용은 **템플릿 삽입으로 허용**"
- 스토어/마케팅에서는 절대 특정 아이돌 이름 사용 금지 유지

### 5. 단계적 구현 로드맵

| 단계 | 범위 | 추정 | 운영 부담 |
|------|------|------|----------|
| **v1.1** (MVP+1~2주) | 마이 아이돌 선택 + 정적 캘린더 활용 (생일/기념일 알림) + 범용 문구 매칭 | 2~3일 | 0분/월 |
| **v1.2** (MVP+4~6주) | 템플릿 문구 + 월간 문구 팩 (수동/반자동) | 3~4일 | 1h/월 |
| **v1.3+** (안정화 후) | Claude CLI 자동 수집 + 문구 생성 + 30개 그룹 확장 | 5~7일 | 30min/월 |
