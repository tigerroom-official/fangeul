## 1. 개요

Sprint 2(상황태그 + K-pop 캘린더 + 분석 계측) 위에 마이 아이돌 선택 + 템플릿 문구 시스템을 통합하여 MVP를 완성한다.

## 3. 온보딩 플로우

**1단계 + 인라인 툴팁** 방식 (패널 절충안 채택):

```
첫 실행 → 그룹 선택 1화면 (건너뛰기="나중에 설정하기")
       ↓
     홈 화면 (인사말 한 줄)
       ↓
     첫 변환 시 툴팁 1줄
       ↓
     버블 Permission 요청 시 툴팁 1줄
```

- 단일 그룹 선택 (멀티스탠은 Phase 7+)
- 5그룹 사전 리스트 + "기타(직접 입력)" 탈출구
- "설정에서 언제든 바꿀 수 있어요" 하단 고정 노출
- 건너뛰기 시 마이 아이돌 = null (템플릿 문구 미노출)

## 4. 마이 아이돌이 앱 전체에 미치는 영향

### 4.1 홈 화면
- 인사말 한 줄: "안녕하세요, {group_name} 팬님!"
- 이벤트 인라인 칩, 데일리 카드 가중 풀 → v1.2로 이동

### 4.2 문구 화면
- "내 아이돌" 필터 칩 좌측 고정 (설정 유저만)
- 미설정 유저: 기존 팩 필터만 표시

### 4.3 캘린더
- 마이 아이돌 필터 기본값 연동 (설정 유저)
- 미설정 유저: 전체 이벤트 표시 (현재와 동일)

### 4.4 컬러 테마
- **공식 그룹 컬러 앱 UI 직접 적용 금지** (IP + rules 준수)
- 감성 컬러 팩 소프트 추천만: `groupToPackHint` 매핑 (BTS → "퍼플 드림")

### 4.5 버블 (미니변환기)
- "오늘" 필터 칩 추가 (설정 유저만, 0dp 추가)
- 미설정 유저: 칩 자체 숨김
- cross-engine sync 필수 (`prefs.reload()` + invalidate)

## 5. 템플릿 문구 시스템

### 5.1 데이터 모델
- `Phrase` entity에 `isTemplate: bool` 필드 추가 (기본값 false)
- 별도 `TemplatablePhrase` entity 생성하지 않음
- 즐겨찾기 키 = 원본 템플릿 문자열 (`{{group_name}} 컴백 축하해요!`)

### 5.2 치환 로직
- 치환 시점: Provider 레이어 (`filteredCompactPhrasesProvider` 파생)
- 위젯은 이미 치환된 `Phrase.ko`를 표시만 함
- 복사 시 치환된 결과가 클립보드에 복사됨

### 5.3 미설정 유저 처리
- `isTemplate == true` 문구를 Provider에서 결과 리스트에서 제외
- UI에 `{{group_name}}` 텍스트 절대 노출하지 않음

### 5.4 표시 방식
- 별도 "내 아이돌" 필터 칩으로 분리 (기존 팩에 혼합하지 않음)
- 칩 좌측 고정, solid fill 강조

### 5.5 IP 방어
- KpopEvent의 artist/group 필드 = 매칭 로직 전용, UI 텍스트 미노출
- 템플릿은 "유저가 선택한 결과 표시" — 앱이 능동적으로 이름 서페이싱하지 않음
- 스토어/마케팅에서 아이돌 이름 절대 사용 금지 유지

## 6. 구현 의존 순서

```
Layer 1 (병렬): MyIdolNotifier + groups.json + Phrase.isTemplate + rules 수정
Layer 2 (병렬): 치환 로직 + 즐겨찾기/필터링 + 선택 UI + 인사말 + 캘린더 연동
Layer 3 (병렬): 필터 칩 + cross-engine sync
Layer 4:        버블 "오늘" 칩
Layer 5:        통합 테스트
```
