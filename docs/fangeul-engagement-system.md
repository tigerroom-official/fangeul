# Fangeul — Engagement & Retention 시스템 설계서

> 유틸리티 앱을 "매일 여는 K-pop 한국어 게임 + 유틸리티"로 전환하는 설계
>
> 최종 업데이트: 2026-02-27

---

## 1. 설계 철학

### 왜 필요한가

현재 MVP 기능(키패드, 변환기, 문구 라이브러리)만으로는 "한번 쓰고 삭제" 앱이 될 확률이 높다. 문구 몇 개 복사하면 클립보드에 남아서 앱을 다시 열 이유가 없어진다.

검증된 모델(Wordle, Duolingo, Snapchat, NYT Games, 가챠 시스템)에서 추출한 리텐션 메커니즘을 K-pop 팬 맥락에 맞게 재설계하여, **DAU 드라이버**를 만든다.

### 핵심 원칙

1. **제한이 중독을 만든다** — Wordle처럼 하루 1판만. 더 하고 싶은데 못 하면 내일 또 온다.
2. **손실 회피가 습관을 만든다** — Duolingo처럼 스트릭을 잃는 공포가 매일 복귀하게 한다.
3. **공유가 바이럴을 만든다** — Wordle 이모지 그리드처럼, 결과를 공유하고 싶게 만들면 유기적 유입이 생긴다.
4. **수집 욕구가 체류를 만든다** — K-pop 포카 문화 + 가챠 메커니즘. 팬들은 이미 수집에 익숙하다.
5. **팬 활동 = 하고 싶은 것** — 학습 동기보다 팬 동기가 강하다. "한국어 공부하세요"가 아니라 "아이돌에게 한 마디 해볼까?"

## 2. 데일리 엔진: 오늘의 한글 퍼즐 (Daily Hangul)

### 2.1 컨셉

Wordle의 K-pop 한국어 버전. **하루 1판, 전 세계 동일 문제, 결과 공유 가능.**

### 2.2 게임 모드

#### Mode A: 한글 Wordle (기본)

- 5~6글자 한국어 단어를 6번 안에 맞추기
- 각 시도마다 피드백:
  - 🟩 초록: 해당 자리에 맞는 글자
  - 🟨 노랑: 단어에 포함되지만 자리가 틀림
  - ⬜ 회색: 단어에 없는 글자
- 입력은 앱 내 한글 키패드 사용 (자연스럽게 키패드 사용 유도)
- 단어 풀: K-pop 관련 단어, 팬 표현, 일상 한국어 (1000개+ 사전 준비)
- 예시: `사랑해` `컴백해` `화이팅` `연습생` `데뷔하` `무대위`

#### Mode B: 로마자 → 한글 맞추기 (보조)

- 로마자 발음이 주어지면 올바른 한글 선택 (4지선다)
- 예: "saranghaeyo" → ① 사랑해요 ② 살아해요 ③ 사항해요 ④ 사랑해용
- 하루 3문제, 30초 타이머
- 기존 문구 라이브러리 데이터로 자동 생성 가능

#### Mode C: K-pop 한글 커넥션 (주간 보너스)

- NYT Connections 모델
- 8개 한국어 단어 → 2그룹으로 분류
- 예: {밥, 김치, 라면, 떡볶이} = 음식 / {사랑, 보고싶다, 행복, 슬픔} = 감정
- 매주 월요일 1판 (주간 보너스 이벤트)

### 2.3 결과 공유 (바이럴 엔진)

```
🇰🇷 Fangeul Daily #127 🔥 4/6

⬜🟨⬜🟩⬜
⬜🟩🟨🟩⬜
🟩🟩⬜🟩🟨
🟩🟩🟩🟩🟩

fangeul.app 💜
```

- "공유" 버튼 → 클립보드 복사 → Twitter/Instagram에 붙여넣기
- K-pop 팬 타임라인에 이 패턴이 돌기 시작하면 자연스러운 바이럴
- **스포일러 없이 결과만 공유** → 궁금증 유발 → 앱 설치 유도

### 2.4 데이터 소스 & 관리

- 단어 풀 JSON: `assets/daily/word_pool.json` (1000개+ 사전 빌드)
- 일일 단어 결정: 솔트 + 해시 기반 인덱스 (전 세계 동일 문제 보장)
- 단어 카테고리: 팬 표현, K-pop 용어, 음식, 감정, 일상, 인사
- **서버 불필요** — 완전 온디바이스 로직

## 3. 스트릭 시스템 (Hangul Streak)

### 3.1 근거

Duolingo 내부 데이터: 7일 이상 스트릭 유저는 3배 더 자주 복귀. 스트릭 푸시 알림이 engagement를 25% 올림. 유저 리텐션 12% → 55%로 개선된 핵심 메커니즘.

### 3.2 스트릭 적립 조건

하루에 아래 중 **하나만** 하면 스트릭 유지:

- 데일리 퍼즐 도전 (성공/실패 무관, 도전 자체가 조건)
- 오늘의 문구 확인 (탭 한 번)
- 로마자 퀴즈 1문제 풀기

의도적으로 허들을 낮춰서 "앱 열고 1탭이면 스트릭 유지" → 습관 형성 우선.

### 3.3 스트릭 보상 체계

| 스트릭 | 보상 |
|---|---|
| 3일 | 한글 카드 1장 보너스 |
| 7일 | 컬러 테마 1종 해금 |
| 14일 | 문구팩 1종 해금 (보상형 광고 대체) |
| 30일 | Pro 기능 3일 무료 체험 |
| 50일 | 특별 뱃지 + 앱 아이콘 변경권 |
| 100일 | 레전드 뱃지 + 전체 테마 해금 |

### 3.4 스트릭 프리즈 (Duolingo 모델)

- 스트릭 프리즈 아이템: 보상형 광고 시청 또는 퀴즈 포인트 소모로 획득
- 사용 시 하루 건너뛰어도 스트릭 유지
- 최대 보유 2개 → 스트릭 길어질수록 프리즈 보관 전략도 생김
- **Pro 유저는 자동 프리즈 1개/주 제공** → Pro 전환 동기

### 3.5 스트릭 UI

- 홈 화면 상단에 🔥 아이콘 + 연속일수 상시 표시
- 스트릭 위험 시 (당일 미완료) 홈 화면에 빨간 점 경고
- 스트릭 달성 시 축하 애니메이션 + 햅틱 피드백

## 4. 킬러 푸시 알림 시스템

### 4.1 왜 K-pop 맥락이 중요한가

일반 학습앱 알림: "오늘 공부 안 했어요!" → 죄책감 → 앱 삭제.
K-pop 팬 맥락 알림: "오늘 한국어 한 마디 배워서 아이돌한테 보내볼까? 💜" → 설렘 → 앱 열기.

### 4.2 알림 시나리오

#### 일일 리마인더 (오후 7~9시, 팬 활동 피크 타임)

- "오늘의 한글 퍼즐 아직 안 풀었어! 🧩 30초면 끝나~"
- "네 🔥 {N}일 스트릭이 위험해! 오늘 안에 한 번만 들어와 💜"
- "새로운 K-pop 문구가 도착했어 ✉️ 오늘 아이돌한테 보내볼까?"

#### 이벤트 연동 알림 (캘린더 기반)

- "{아이돌명} 생일 D-3! 🎂 생일 축하 문구 미리 준비해둘까?"
- "{그룹명} 컴백 D-1! 🎵 컴백 축하 문구팩 미리보기 →"
- "오늘 {그룹명} 음방 출연일! 📺 응원 문구 복사해서 실시간 채팅에 쓰자!"

#### 스트릭 긴급 알림 (오후 10시, 마지막 기회)

- "⚠️ 스트릭 마감 2시간 전! 퍼즐 한 판이면 유지돼~"
- "🔥 {N}일 스트릭 끊기면 아깝잖아! 지금 10초만 →"

### 4.3 알림 정책

- 최대 하루 2회 (과도한 알림 → 삭제 유발)
- 첫 주: 매일 알림 → 2주차: 격일 → 3주차 이후: 스트릭 위험시에만
- 유저가 알림 빈도 직접 설정 가능 (설정 > 알림 > 매일/가끔/이벤트만)
- 알림 열림률 추적 → A/B 테스트로 문구 최적화

## 5. 한글 카드 컬렉션 (Hangul Cards)

### 5.1 컨셉

K-pop 포카(포토카드) 수집 문화 + 게임 가챠 시스템. 매일 한국어 단어 카드를 수집하여 카테고리별 도감을 완성한다.

### 5.2 카드 구성

각 카드:
- 앞면: 한글 단어 + 카테고리 아이콘 + 카드 등급 색상 + 감성 배경
- 뒷면: 로마자 발음 + 영어/모국어 뜻 + 사용 맥락 + 🔊 TTS
- 등급: ⭐Common / ⭐⭐Rare / ⭐⭐⭐Epic (출현 확률로 구분)

### 5.3 카드 획득 방식

| 방법 | 획득량 |
|---|---|
| 매일 첫 앱 오픈 (출석 보상) | 1장 |
| 데일리 퍼즐 클리어 | 1장 |
| 스트릭 보상 (3일, 7일 등) | 보너스 1~3장 |
| 보상형 광고 시청 | 1장 |
| 친구 초대 | 3장 |

### 5.4 카드 도감 (컬렉션 보상)

| 카테고리 | 필요 카드 수 | 완성 보상 |
|---|---|---|
| 🍜 음식 (밥, 김치, 떡볶이...) | 10장 | 음식 문구팩 해금 |
| 💜 사랑 (사랑, 좋아해, 보고싶다...) | 10장 | 사랑 테마 해금 |
| 🎤 K-pop 용어 (컴백, 음방, 팬싸...) | 15장 | K-pop 용어 사전 해금 |
| 🎉 축하 (생일, 데뷔, 1위...) | 10장 | 축하 문구팩 해금 |
| 😊 감정 (행복, 슬픔, 화이팅...) | 10장 | 감정 이모지 테마 해금 |
| 🌟 레전드 (전체 도감 80%) | 전체 | 레전드 프로필 프레임 |

### 5.5 중복 카드 처리

- 이미 가진 카드 획득 시 → "별가루" 포인트로 전환
- 별가루 N개 → 원하는 카드 1장 지정 교환 (너무 쉽지 않게 밸런스)
- Pro 유저: 중복 카드 출현 확률 50% 감소

### 5.6 카드 비주얼 디자인 & 생성 파이프라인

#### 디자인 원칙

아이돌 초상/캐릭터/로고는 **일절 사용하지 않는다** (초상권 + 상표권 리스크).
대신 **"한글 타이포그래피 + 감성 일러스트 배경"** 조합으로, 아이돌 없이도 수집하고 싶은 카드를 만든다.

K-pop 팬이 이미 수집하는 것: 폰 배경화면, 타이포 짤, 감성 카드, 팬메이드 포스터.
이 문화에 맞춰 **"갤러리에 저장하고 싶은 퀄리티"**를 기준으로 한다.

#### 카드 레이아웃

앞면:
```
┌─────────────────────┐
│  [카테고리 아이콘]  ⭐⭐ │  ← 등급 표시
│                     │
│    감성 일러스트      │  ← ComfyUI 생성 배경
│      배경 영역       │
│                     │
│   ┌─────────────┐   │
│   │  사랑해요    │   │  ← 한글 타이포그래피 (메인)
│   └─────────────┘   │
│    saranghaeyo      │  ← 로마자 (서브)
│                     │
│  ─── Fangeul ───    │  ← 워터마크
└─────────────────────┘
```

뒷면:
```
┌──────────────────────┐
│  사랑해요             │  ← 한글
│  saranghaeyo          │  ← 로마자 발음
│  🇬🇧 I love you       │  ← 영어
│  🇮🇩 Aku cinta kamu   │  ← My Language
│                       │
│  💬 "아이돌 생일에     │  ← 사용 맥락
│     진심을 전할 때"    │
│                       │
│  🔊 발음 듣기         │  ← TTS 재생 버튼
└──────────────────────┘
```

#### 등급별 비주얼 차이

| 등급 | 배경 | 타이포 | 특수 효과 |
|---|---|---|---|
| ⭐ Common | 단색 그라데이션 (파스텔톤) | 기본 산세리프 폰트 | 없음 |
| ⭐⭐ Rare | ComfyUI 감성 일러스트 (벚꽃, 별, 노을, 네온) | 캘리그래피 스타일 | 미세 파티클 애니메이션 |
| ⭐⭐⭐ Epic | ComfyUI 프리미엄 일러스트 (홀로그램, 오로라, 우주) | 3D 입체 타이포 | 홀로그램 반짝임 + 터치 인터랙션 |

#### ComfyUI 생성 파이프라인

```
[카드 기획 데이터 JSON]
  ↓ 카테고리 + 감성 키워드 추출
[ComfyUI 워크플로우 — RTX 5090]
  ├─ 체크포인트: waiIllustriousSDXL (고품질 일러스트)
  ├─ 프롬프트 구조:
  │   Positive: "beautiful aesthetic background, {category_keyword},
  │    pastel colors, dreamy atmosphere, no text, no people,
  │    no characters, illustration style, high quality"
  │   Negative: "person, face, character, idol, celebrity,
  │    realistic photo, text, watermark, logo"
  ├─ 해상도: 768x1024 (카드 비율 3:4)
  ├─ 후처리: 한글 타이포 오버레이 (Pillow/ImageMagick)
  │   - Common: 시스템 폰트 (Pretendard 등)
  │   - Rare: 무료 캘리폰트 (배민 한나체, 나눔 손글씨 등)
  │   - Epic: 커스텀 3D 렌더 타이포
  └─ 워터마크: 하단 중앙 "Fangeul" (투명도 40%)
  ↓ 배치 생성 (카드 200~300장)
[품질 검수 — Claude Vision 자동 QA]
  ├─ 사람/캐릭터 포함 여부 체크 (reject if detected)
  ├─ 텍스트 가독성 체크
  └─ 전체적 심미성 스코어링
  ↓ 통과된 카드만
[최적화: WebP 변환, 용량 압축 (카드당 50~100KB)]
  ↓ 업로드
[Cloudflare R2]
  ↓ 앱에서 다운로드
[로컬 캐시]
```

#### 생성 규모 & 운영

- 초기: 200장 (카테고리 5개 × 등급 3개 × ~13장)
- Common 120장 / Rare 60장 / Epic 20장
- 카드 이미지 총 용량: 약 15~25MB (WebP 압축 시)
- Common은 앱 번들에 포함 (오프라인 즉시 사용)
- Rare/Epic은 R2에서 다운로드 (획득 시 자동)
- 신규 카드 추가: ComfyUI 워크플로우 1회 실행 → R2 업로드 → 앱에서 새 카드 감지
- **시즌/이벤트 한정 카드:** 벚꽃 시즌, 크리스마스, 설날 등 → 한정 수집 욕구 자극

#### 카드 저장 & 공유 (바이럴)

- "갤러리에 저장" 버튼 → 고해상도 PNG로 폰에 다운로드
- "공유" 버튼 → Twitter/Instagram 스토리에 직접 공유
- 공유 시 카드 하단 워터마크 + "Fangeul에서 한국어 카드 모으기 💜" 텍스트 자동 첨부
- 팬 계정이 예쁜 카드를 공유하면 → 자연스러운 앱 홍보 + 유입

## 6. K-pop 캘린더 + 문구 연동

### 6.1 컨셉

Blip 앱처럼 아이돌 스케줄 캘린더를 제공하되, Fangeul만의 차별화는 **"이 날 보낼 한국어 메시지"를 바로 연결**하는 것.

### 6.2 캘린더 데이터

#### 수집 대상

| 데이터 | 소스 | 수집 방식 |
|---|---|---|
| 멤버 생일 | K-pop 프로필 DB (공개 정보) | 초기 JSON 빌드, 수동 업데이트 |
| 그룹 데뷔 기념일 | 공개 정보 | 초기 JSON 빌드 |
| 컴백 스케줄 | Blip, @KPOPSchedule 계정, 팬사이트 | 주기적 크롤링 (주 2~3회) |
| 음방 출연 | 각 음악방송 공식 SNS | 주기적 크롤링 |
| 콘서트/팬미팅 | 공식 공지, 티켓팅 사이트 | 주기적 크롤링 |

#### 크롤링 파이프라인

```
[데이터 소스] → [RTX 5090 서버 크롤러 (Python/Scrapy)]
     ↓ 주 2~3회 배치
[JSON 가공 + 검증]
     ↓ 업로드
[Cloudflare R2 버킷] → schedule_v{version}.json
     ↓ 앱에서 버전 체크
[앱 → 새 데이터 감지 → 다운로드 → 로컬 캐시]
```

- 앱은 하루 1회 R2에서 버전 체크 → 새 데이터 있으면 백그라운드 다운로드
- 오프라인에서도 기존 캐시 데이터로 캘린더 표시
- 초기 MVP는 상위 30개 그룹만 커버 → 유저 피드백 보고 확장

## 7. Flash Phrase (BeReal 모델)

### 7.1 컨셉

BeReal의 "랜덤 시간 알림 + 제한 시간 응답" 모델 적용. 하루 중 랜덤한 시간에 푸시 → 30분 내 문구 퀴즈 풀면 보너스 카드.

### 7.2 동작 방식

1. 하루 1회, 오전 10시~오후 9시 사이 랜덤 시간에 푸시 알림
2. "⚡ Flash Phrase! 30분 안에 풀면 보너스 카드!"
3. 앱 열면 한국어 문구 퀴즈 1문제 (예: "이 로마자의 한글은?" 4지선다)
4. 30분 내 정답 → 보너스 카드 1장 + 스트릭 유지
5. 시간 초과 → 아무 패널티 없음 (FOMO만 유발, 벌칙 없음)

### 7.3 구현

- 로컬 알림 스케줄링 (AlarmManager)
- 전날 밤 다음날 Flash Phrase 시간 랜덤 결정 → 암호화 저장
- 완전 옵셔널 기능 (설정에서 끄기 가능)
- 시간 조작 방어: 알림 수신 시간과 앱 오픈 시간 교차 검증

## 8. 보안 & 어뷰징 방어

### 8.1 위협 모델

| 위협 | 방법 | 영향 |
|---|---|---|
| APK 디컴파일 → 단어 풀 추출 | apktool, jadx 등 | 퍼즐 답 유출, 스포일러 확산 |
| SharedPreferences 조작 | root + 파일 편집 | 스트릭 무한, 카드 무한 획득 |
| 시간 조작 (기기 시계 변경) | 설정 > 날짜/시간 수동 변경 | 하루 여러 번 데일리 보상 수령 |
| 메모리 해킹 | GameGuardian, Cheat Engine | 포인트/카드 수 변조 |
| 보상형 광고 콜백 위조 | 네트워크 패킷 조작 | 광고 안 보고 보상 수령 |

### 8.2 방어 전략

#### 데일리 퍼즐 답 보호

- 단어 풀 자체는 APK에 포함하되 AES-256 암호화
- 복호화 키는 앱 코드 내 분산 저장 (단일 문자열 X), 런타임에만 조합
- 일일 인덱스: 예측 가능한 단순 모듈로 연산 대신 솔트+해시 기반

```dart
// ✅ 솔트 + 해시 기반 인덱스 (예측 불가)
final seed = sha256('fangeul_daily_${dateString}_${appSecret}');
final index = seed.hashCode.abs() % wordPool.length;
```

#### 로컬 데이터 무결성

- `flutter_secure_storage` 사용 (Android Keystore 기반 암호화)
- 각 데이터에 HMAC 서명 추가 → 로드 시 서명 검증 → 불일치 시 초기화

```dart
// 저장 시
final data = jsonEncode({'streak': 15, 'cards': [...]});
final hmac = Hmac(sha256, appKey).convert(utf8.encode(data));
await secureStorage.write(key: 'game_state', value: data);
await secureStorage.write(key: 'game_state_sig', value: hmac.toString());

// 로드 시: 서명 불일치 → 데이터 변조 감지 → 초기화
```

#### 시간 조작 방어

- 단조증가 타임스탬프: 마지막 데일리 완료 시간을 암호화 저장
- 현재 시간이 마지막 완료 시간보다 과거 → 시간 조작 판정
- 48시간 이상 미래 점프 → 1일치만 인정
- 네트워크 가능 시 R2 서버 시간과 기기 시간 교차 검증 (차이 > 5분 → 차단)

#### 보상형 광고 검증

- AdMob SDK `onUserEarnedReward` 콜백 + 중복 지급 방지
- 같은 종류 보상 5분 내 재수령 불가
- 하루 보상형 광고 시청 최대 10회 제한
- 향후 서버 구축 시 AdMob SSV (Server Side Verification) 적용

#### APK 무결성

- 릴리즈 빌드 ProGuard/R8 코드 난독화
- 런타임 APK 서명 해시 검증 (리패키징 감지)
- Play Integrity API로 root 감지 → root 기기에서 보상 기능만 비활성화

### 8.3 방어 우선순위 (MVP)

| 우선순위 | 방어 항목 | 구현 난이도 |
|---|---|---|
| P0 (필수) | 로컬 데이터 HMAC 서명 | 낮음 |
| P0 (필수) | 시간 조작 감지 (단조증가) | 낮음 |
| P0 (필수) | 보상형 광고 중복 지급 방지 | 낮음 |
| P1 (중요) | 단어 풀 AES 암호화 | 중간 |
| P1 (중요) | ProGuard/R8 난독화 | 낮음 (빌드 설정) |
| P2 (후순위) | APK 서명 런타임 검증 | 중간 |
| P2 (후순위) | Play Integrity API (root 감지) | 중간 |
| P3 (나중) | 서버 사이드 광고 검증 (SSV) | 높음 (서버 필요) |
